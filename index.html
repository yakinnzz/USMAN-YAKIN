1. PENGERTIAN OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang menggunakan objek sebagai dasar perancangan program. Pendekatan ini memungkinkan kode menjadi lebih terstruktur, modular, dan mudah dikembangkan.

2. Class Dan Object

Class merupakan blueprint atau cetak biru yang digunakan untuk membuat objek. Class berisi atribut (variabel) yang menggambarkan karakteristik suatu objek serta method (fungsi) yang menggambarkan perilaku objek tersebut. Sedangkan object merupakan instansi dari sebuah class.

3. CONSTRUCTOR

Constructor adalah method khusus untuk inisialisasi nilai awal pada objek ketika objek dibuat.

class Car {
    String? model;
    String? color;
    
    Car(this.model, this.color);
    
    void start() { 
        print('Car started!'); 
    }
}

4. VALIDASI DATA

Validasi data dalam constructor memastikan data yang dimasukkan sesuai aturan.

class Mahasiswa {
    String nama;
    String nim;
    double ipk;
    
    Mahasiswa(this.nama, this.nim, this.ipk) {
        if (ipk < 0.0 || ipk > 4.0) {
            throw ArgumentError('IPK harus 0.0 - 4.0');
        }
    }
}

5. ENCAPSULATION

Encapsulation membungkus data dan method serta membatasi akses langsung dari luar class.

class Person {
    String _name;
    int age;
    
    Person(this._name, this.age);
    
    void setName(String name) { _name = name; }
    String getName() { return _name; }
}

6. NAMED PARAMETER

Named parameter memungkinkan parameter dipanggil berdasarkan nama, membuat kode lebih jelas.

class Produk {
    final String id;
    final String nama;
    final double harga;
    final int stok;
    final String kategori;
    
    Produk({
        required this.id,
        required this.nama,
        required this.harga,
        this.stok = 0,
        this.kategori = 'Umum',
    });
}

7. KESIMPULAN

Konsep OOP seperti class, object, constructor, validasi data, encapsulation, dan named parameter membuat program lebih terstruktur, mudah dipelihara, dan dapat dikembangkan menjadi sistem yang lebih kompleks.
