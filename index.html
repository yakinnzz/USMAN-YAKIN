1. PENGERTIAN OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang menggunakan objek sebagai dasar perancangan program. Pendekatan ini memungkinkan kode menjadi lebih terstruktur, modular, dan mudah dikembangkan.

2. Class Dan Object

Class merupakan blueprint atau cetak biru yang digunakan untuk membuat objek. Class berisi atribut (variabel) yang menggambarkan karakteristik suatu objek serta method (fungsi) yang menggambarkan perilaku objek tersebut. Dengan kata lain, class menjadi kerangka dasar yang menentukan bagaimana suatu objek akan dibuat dan bekerja. Sedangkan object merupakan instansi dari sebuah class. Object adalah representasi nyata dari class yang telah dibuat dan dapat memiliki nilai atribut yang berbeda-beda. Jika class diibaratkan sebagai cetakan, maka object adalah hasil dari cetakan tersebut. 


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

3. CONSTRUCTOR

Constructor adalah sebuah method khusus yang terdapat di dalam class yang digunakan untuk melakukan inisialisasi nilai awal pada sebuah objek ketika objek tersebut dibuat. Constructor biasanya memiliki nama yang sama dengan nama class dan akan otomatis dipanggil saat proses pembuatan objek berlangsung. 


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

4. VALIDASI DATA

Constructor adalah sebuah method khusus yang terdapat di dalam class yang digunakan untuk melakukan inisialisasi nilai awal pada sebuah objek ketika objek tersebut dibuat. Constructor biasanya memiliki nama yang sama dengan nama class dan akan otomatis dipanggil saat proses pembuatan objek berlangsung. 


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus 0.0 - 4.0');
    }
  }
}

5. ENCAPSULATION

Encapsulation merupakan salah satu konsep utama dalam Object Oriented Programming (OOP) yang berfungsi untuk membungkus data dan method dalam sebuah class serta membatasi akses langsung terhadap data tersebut dari luar class. 


class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

6. NAMED PARAMETER

Named parameter merupakan salah satu fitur dalam bahasa pemrograman Dart yang memungkinkan parameter dalam sebuah constructor atau fungsi dipanggil berdasarkan nama parameternya. Dengan menggunakan named parameter, kode program menjadi lebih jelas, mudah dibaca, dan lebih fleksibel dalam penggunaannya. 


class Produk {
  final String id;
  final String nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

7. KESIMPULAN

Beberapa konsep penting dalam OOP yang telah dipelajari antara lain class dan object yang berfungsi sebagai dasar pembentukan program, constructor yang digunakan untuk memberikan nilai awal pada objek, validasi data yang membantu memastikan bahwa data yang dimasukkan sesuai dengan aturan yang ditentukan, serta encapsulation yang berfungsi untuk melindungi data agar tidak dapat diakses secara langsung dari luar class. Selain itu, penggunaan named parameter dalam Dart juga memberikan fleksibilitas dan kemudahan dalam proses pembuatan objek.

OOP membuat program lebih terstruktur, mudah dipelihara, dan dapat dikembangkan menjadi sistem yang lebih kompleks.
